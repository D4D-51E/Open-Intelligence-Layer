## 0. 핵심 요약

현재 가정:

- 2인 팀
- AI/SW 중심
- 별도 드론·로봇·센서 하드웨어는 없거나 제한적
- 24시간 안에 데모 가능한 결과물이 중요
- 3분 발표에서 문제의식, 데모, 국방 적용성을 빠르게 보여줘야 함

### 1차 추천

**1순위: T1 - CCA Mission Allocation Sandbox**  
한국어로는 **CCA 임무배분 샌드박스**.

한 줄 설명:

> 조종사/임무통제관이 여러 무인기를 직접 하나하나 조종하지 않고, 임무 의도만 입력하면 AI가 ISR, EW, Decoy, Sensor Extension 역할을 자동 배분하고 상황 변화 시 재배분하는 시뮬레이터.

추천 이유:

- 하드웨어 없이 공개·가상 데이터만으로 구현 가능
- 국방/방산 맥락이 강함
- 단순 챗봇이 아니라 **시뮬레이션 + 자동 배분 알고리즘 + AI 브리핑**으로 차별화 가능
- 3분 발표 데모 흐름이 직관적임
- “인지부하 감소”라는 평가 포인트가 명확함

### 플랜 B

**2순위: T3 - Link-16 Concept COP Simulator**  
가상 전술 메시지를 공통작전상황도(COP)로 보여주고 자연어 브리핑을 생성하는 시뮬레이터.

**3순위: T5 - War-game AI Tutor**  
장교/간부 훈련용 AI 워게임 교관. 구현은 쉽지만 흔한 LLM 챗봇처럼 보이지 않게 해야 함.

## T1. Autonomy, Unmanned Systems & Counter-UAS

쉽게 말하면:

> 드론, 무인기, 대드론, 유무인복합 작전을 더 적은 사람이 더 똑똑하게 운용하게 만드는 트랙.

T1은 국방색이 강하고 심사위원에게 임팩트가 잘 전달될 가능성이 높다. 다만 실제 하드웨어가 필요한 문제는 2인팀에게 부담이 크다.

| 후보 문제 | 추천도 | 쉬운 설명 | 이유 | 리스크 |
|---|---:|---|---|---|
| 1인 다중 무인기 동시 통제 | 중상 | 한 명이 여러 드론/로봇을 동시에 관리하는 시스템 | 임무 단위 UI, 자동 역할 배분, 인지부하 감소를 보여주기 좋음 | 실제 무인기 없이 하면 시뮬레이션 설득력이 필요 |
| 산악·도심 혼재 지형 자율 드론 운용 | 중 | 산악/도심 지형을 고려해 드론 경로를 똑똑하게 짜는 문제 | 3D 경로계획, 통신 단절 회피 등 기술적 깊이가 있음 | 지형 데이터·3D 시각화·경로 알고리즘 범위가 커질 수 있음 |
| 수도권 영공 소형드론·풍선 침투 대응 | 중상 | 도심 상공의 수상한 드론/풍선을 탐지·위협평가하는 시스템 | 한국적 시나리오가 강하고 대시보드 데모가 좋음 | 실제 센서 데이터가 없으면 synthetic 데이터 설계가 중요 |
| **CCA 임무배분 샌드박스** | **상** | 유인기와 여러 무인기 사이 임무를 자동 배분하는 시뮬레이터 | 하드웨어 없이 가능, 국방색 강함, 알고리즘+UI+AI 브리핑 조합이 좋음 | 공중전 도메인 용어를 너무 과하게 쓰면 이해가 어려워질 수 있음 |
| 자율 요격 드론 Mini Operational Test | 중하/하 | 실내에서 요격 드론 테스트를 실제로 수행하는 문제 | 하드웨어가 있으면 매우 강력하고 눈에 띔 | 드론 장비, 안전, 제어, 현장 테스트 리스크가 큼. 사전 연락 필요 |
| 무선침묵 환경 피아식별(IFF) | 중 | 전파를 거의 내지 않고 아군/적 무인기를 식별하는 체계 | 암호학·보안 쪽으로 차별화 가능 | 시각적 데모가 약할 수 있고 국방 프로토콜 설명이 어려울 수 있음 |

### T1 내 추천 순위

1. **CCA 임무배분 샌드박스**
2. 1인 다중 무인기 동시 통제
3. 수도권 영공 소형드론·풍선 침투 대응
4. 무선침묵 환경 피아식별
5. 산악·도심 혼재 지형 자율 드론 운용
6. 자율 요격 드론 Mini Operational Test

## T2. OSINT & Defense Intelligence

쉽게 말하면:

> 인터넷, 뉴스, 항적, 공개 데이터, 유출 징후 같은 공개정보를 모아 군사·안보 분석가가 쓸 수 있는 정보 시스템을 만드는 트랙.

T2는 AI/LLM/RAG 팀에게 잘 맞는다. 다만 “그냥 검색 챗봇”처럼 보이면 약하므로, 출처·신뢰도·교차검증·시각화가 중요하다.

| 후보 문제 | 추천도 | 쉬운 설명 | 이유 | 리스크 |
|---|---:|---|---|---|
| 멀티소스 융합 인텔 코파일럿 | 중상 | 여러 공개정보 소스를 검색하고 출처와 함께 답하는 정보분석 코파일럿 | LLM/RAG 역량을 잘 보여줄 수 있고 구현 난이도도 적절함 | 흔한 RAG 챗봇처럼 보일 위험. 출처/신뢰도/시각화가 필수 |
| Maven형 공중 ISR 융합 코파일럿 | 중상 | 항적, 위성궤도, 기상, 뉴스 등을 합쳐 공중·우주 이상징후를 찾는 시스템 | 지도/타임라인 데모가 좋고 공중 트랙 색깔이 강함 | 공개 항적/궤도 데이터 연결이 시간이 걸릴 수 있음 |
| 방산 공급망 자격증명 노출 조기경보 | 중 | 방산 협력사 도메인의 유출 계정/위험을 조기경보하는 시스템 | 사이버보안 색깔이 강하고 비즈니스 임팩트가 명확함 | 실제 유출 데이터 사용은 윤리/법적 이슈가 있어 mock 데이터 설계 필요 |
| 위협 행위자 지식그래프 & 엔티티 해소 | 중 | 별칭, 이메일, 지갑주소 등을 묶어 위협행위자 관계 그래프를 만드는 시스템 | 그래프 시각화와 entity resolution이 차별화 포인트 | 고품질 데이터 없으면 데모가 인위적으로 보일 수 있음 |

### T2 내 추천 순위

1. Maven형 공중 ISR 융합 코파일럿
2. 멀티소스 융합 인텔 코파일럿
3. 위협 행위자 지식그래프 & 엔티티 해소
4. 방산 공급망 자격증명 노출 조기경보

## T3. Battle Network, C2 & Sustainment

쉽게 말하면:

> 전장 상황을 한눈에 보고, 지휘관이 빠르게 판단하고, 통신이 끊겨도 작전을 유지하게 돕는 시스템을 만드는 트랙.

T3는 대시보드·시뮬레이션·자연어 브리핑과 잘 맞는다. 2인팀은 “전장 전체 플랫폼”이 아니라 특정 시나리오 하나로 줄여야 한다.

| 후보 문제 | 추천도 | 쉬운 설명 | 이유 | 리스크 |
|---|---:|---|---|---|
| 멀티모달 센서, OSINT 퓨전 → 자연어 전장 COP | 중 | 여러 센서/OSINT를 합쳐 자연어 질의 가능한 전장 상황도를 만드는 문제 | COP 대시보드와 AI 질의응답이 직관적 | 범위가 매우 넓고 실제 센서 데이터가 부족할 수 있음 |
| Palantir AIP 공중·우주 온톨로지 빌더 | 중상 | 항공기, 위성, 센서, 임무, 위협을 객체/관계로 구조화하는 빌더 | 그래프/온톨로지/LLM 질의응답이 잘 맞음 | Palantir/AIP 느낌을 어떻게 구현할지 스토리 설계 필요 |
| **Link-16 개념 COP 시뮬레이터** | **상** | 가상 전술 메시지를 지도 상황도로 바꾸고 브리핑하는 시스템 | synthetic 데이터로 가능, 데모가 안정적, 발표가 쉬움 | 너무 단순한 지도 대시보드로 보이면 약함 |
| 통합방공망 교전지원 시뮬레이터 | 중상 | 항공기/드론/미사일 위협 우선순위와 방어자산 배정을 시뮬레이션 | 시뮬레이션과 최적화가 명확하고 국방색이 강함 | 교전규칙/방어자산 로직을 단순화해야 함 |
| S-DOT 시맨틱 전송 | 중 | 통신이 제한된 환경에서 중요한 전장 정보만 의미 단위로 전달하는 문제 | 거부환경·전술네트워크 주제가 좋음 | 구현/평가 기준이 추상적일 수 있음 |
| APAC 회색지대 거부환경 대응 동맹군 연합 지속지원 C2 | 중 | 통신/물류가 끊긴 상황에서 동맹군 보급·정비를 유지하는 시스템 | 물류 최적화와 지도 시각화가 잘 맞음 | 문제 범위가 너무 큼. MVP를 보급로 재계산 하나로 줄여야 함 |

### T3 내 추천 순위

1. **Link-16 개념 COP 시뮬레이터**
2. 통합방공망 교전지원 시뮬레이터
3. Palantir AIP 공중·우주 온톨로지 빌더
4. APAC 회색지대 지속지원 C2
5. 멀티모달 센서/OSINT COP
6. S-DOT 시맨틱 전송

## T4. Maritime Domain Awareness

쉽게 말하면:

> 바다에서 AIS를 끄거나 위장하는 수상한 선박, 해상 민병대, 불법 선단, 회색지대 도발 징후를 찾아내는 트랙.

T4는 해상·AIS·위성·지도 데이터에 익숙하면 좋다. 하지만 2인팀이 처음부터 데이터 수집까지 하려면 시간이 부족할 수 있다.

| 후보 문제 | 추천도 | 쉬운 설명 | 이유 | 리스크 |
|---|---:|---|---|---|
| 비정형 데이터 기반 해상 불법·위장 선박 자동 탐지 | 중 | AIS, SAR, OSINT 등을 합쳐 수상한 선박을 찾는 문제 | 지도 기반 데모와 이상탐지가 잘 맞음 | 실제 해상 데이터와 선박 패턴 지식이 필요 |
| OSINT 기반 해상 회색지대 사전 징후 조기탐지 | 중상 | 선박이 출항하기 전 온라인/물류/항만 징후로 도발 가능성을 예측 | LLM/OSINT/시계열 분석과 잘 맞고 스토리가 좋음 | 신뢰할 만한 공개 데이터 확보가 어려울 수 있음 |
| 해상 환경 적응형 무인 감시 자산 기반 위협 탐지 및 센서 퓨전 | 중하 | 부표/USV 센서로 수상·수중 위협을 탐지하는 문제 | 센서퓨전/무인자산 주제가 강함 | 하드웨어/센서 데이터가 없으면 설득력 확보가 어려움 |

### T4 내 추천 순위

1. OSINT 기반 해상 회색지대 사전 징후 조기탐지
2. 비정형 데이터 기반 해상 불법·위장 선박 자동 탐지
3. 해상 환경 적응형 무인 감시 자산 기반 위협 탐지 및 센서 퓨전

## T5. Force Readiness, Training & Simulation

쉽게 말하면:

> 군인·조종사·지휘관이 더 빨리 배우고 더 나은 판단을 하게 만드는 AI 훈련/시뮬레이션 트랙.

T5는 구현 난이도가 상대적으로 낮고 2인팀에게 안전하다. 대신 흔한 AI 튜터/챗봇으로 보이면 약하므로, 시나리오 시뮬레이션과 평가 루브릭이 필요하다.

| 후보 문제 | 추천도 | 쉬운 설명 | 이유 | 리스크 |
|---|---:|---|---|---|
| AI 워게임 교관 / 전술 의사결정 시뮬레이터 | 중상 | 워게임 상황에서 선택지를 제시하고 결과와 피드백을 주는 AI 교관 | 구현이 현실적이고 발표가 쉬움 | 흔한 챗봇이 되기 쉬움. 반드시 시뮬레이션/점수화 필요 |
| 거부환경 온디바이스 전술 어시스턴트 | 중 | 오프라인/보안망에서 교범·전술 질문에 답하는 전술 AI | 온디바이스/오프라인 컨셉이 명확함 | 실제 온디바이스 최적화까지 하기는 빡셈. 데모 범위 제한 필요 |
| AI Pilot 공중전 의사결정 코치 | 중상 | 공중전/방공회피 상황에서 조종사의 선택지를 비교해주는 코치 | T1/T3와 연결되는 공중전 시나리오가 강함 | 공중전 모델을 너무 복잡하게 잡으면 실패 위험 |
| 징병·단기복무 전술 숙달 가속 시뮬레이터 | 중 | 단기복무 장병이 장비/절차를 빠르게 익히게 하는 개인화 훈련 시스템 | UX와 교육효과를 보여주기 좋음 | 국방 임팩트가 T1/T3보다 약해 보일 수 있음 |

## 5. 최종 추천 전략

### 추천안 A: 차별화형

**T1 - CCA 임무배분 샌드박스**로 간다.

핵심 메시지:

> 미래 공중전에서는 조종사 1명이 여러 CCA/무인기를 동시에 운용해야 한다. 우리는 조종사가 개별 기체를 조종하지 않고 임무 의도만 입력하면, 시스템이 ISR/EW/Decoy/Sensor Extension 역할을 자동 배분하고 위협 변화 시 즉시 재배분하는 샌드박스를 만든다.

MVP 기능:

1. 가상 작전 상황판
2. CCA/무인기 2~4대 상태 표시
3. 임무 자동 배분
4. 위협 발생 시 재배분
5. AI 브리핑/AAR 생성

3분 데모 흐름:

1. 조종사가 “적 방공망을 우회해 목표 지역 ISR 수행” 같은 임무 의도를 입력
2. 시스템이 CCA 4대에 ISR/EW/Decoy/Sensor Extension 임무 자동 배정
3. 위협 발생 또는 기체 손상 이벤트 발생
4. 자동 재배분
5. AI가 “왜 이 배분이 최적인지” 브리핑

---

### 추천안 B: 안정형

**T3 - Link-16 개념 COP 시뮬레이터**로 간다.

핵심 메시지:

> 실제 전술 데이터가 아닌 가상 메시지를 사용해 항공기, 위협, 센서, 임무 정보를 공통작전상황도(COP)로 시각화하고, 지휘관에게 자연어 브리핑을 제공한다.

MVP 기능:

1. 가상 전술 메시지 생성
2. 메시지 파싱
3. 지도/상황판 시각화
4. 트랙 중복 제거 또는 위협도 표시
5. 자연어 브리핑 생성

---

### 추천안 C: 쉬운 완성형

**T5 - AI 워게임 교관 / 전술 의사결정 시뮬레이터**로 간다.

핵심 메시지:

> 훈련자가 전술 상황에서 결심안을 선택하면, AI 교관이 대안, 예상 결과, 교리 기반 피드백, AAR을 제공해 전술 숙달 속도를 높인다.

MVP 기능:

1. 시나리오 생성
2. 사용자 결심 입력
3. 결과 시뮬레이션
4. 점수화/피드백
5. AAR 리포트
