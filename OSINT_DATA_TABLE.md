# OSINT Data Table

_Last updated: 2026-07-04 KST_

This document summarizes the OSINT data currently used by AirMaven Lite. It is based on the live cache schema in `src/lib/types.ts`, the live fetcher in `scripts/fetch-live-data.mjs`, and the current snapshot at `public/data/live-scenarios.json` generated on `2026-07-01T15:35:11.863Z`.

## 1. Current OSINT Layers

| Layer | Dataset key | Current source | Purpose | Map behavior | Caveat |
|---|---|---|---|---|---|
| OSINT article signal | `osint` | `google-news-rss-cache`, fallback from GDELT | Public news/open-source activity signal per AOI | Not directly drawn unless converted to `osintEvents` | News relevance is query-based, not an official incident assessment |
| OSINT review point | `osintEvents` | Derived from `osint` | Clickable map marker for analyst review | Rendered on globe/map as orange OSINT points | Coordinates are AOI review points, not article geocoding |
| Source health | `sourceStatus`, `sourceReasons` | Fetch pipeline metadata | Shows whether GDELT/Google News/derived events are live, cached, rate-limited, etc. | Used in status/diagnostic UI | Status is per snapshot, not permanent service state |
| Cyber/darkweb intel | planned | StealthMole | Planned layer for darkweb, leak, ransomware, and compromised-data signals | Not active yet | API authentication is currently blocked by `401 Unauthorized` |

## 4. `osint` Article Schema

| Field | Type | Required | Description | Example |
|---|---|---:|---|---|
| `id` | `string` | Yes | Stable item ID in the live cache | `live-osint-google-seoul-airspace-1` |
| `source` | `'gdelt-cache' \| 'google-news-rss-cache'` | Yes | Fetch source identifier | `google-news-rss-cache` |
| `regionId` | `RegionId` | Yes | AOI/region this item belongs to | `seoul-airspace` |
| `title` | `string` | Yes | Article or open-source signal title | `Chinese, Russian military aircraft enter KADIZ...` |
| `url` | `string` | No | Source URL or RSS redirect URL | `https://news.google.com/rss/articles/...` |
| `domain` | `string` | No | Publisher/source label | `The Korea Herald` |
| `publishedAt` | ISO datetime string | Yes | Published or observed timestamp | `2026-06-27T07:44:55.000Z` |
| `summary` | `string` | Yes | Fetch context summary generated by the pipeline | `Live Google News RSS search signal for query...` |
| `tags` | `string[]` | Yes | Classification and provenance tags | `google news`, `osint`, `news volume`, `seoul-airspace` |
| `confidence` | `number` | Yes | Lightweight display/scoring confidence, `0.0` to `1.0` | `0.64` |

## 5. `osintEvents` Map Point Schema

| Field | Type | Required | Description | Example / note |
|---|---|---:|---|---|
| `id` | `string` | Yes | Derived map event ID | `osint-review-live-osint-google-seoul-airspace-1` |
| `regionId` | `RegionId` | Yes | AOI/region this marker belongs to | `seoul-airspace` |
| `source` | `OsintItem['source']` | Yes | Source inherited from the related article | `google-news-rss-cache` |
| `title` | `string` | Yes | Marker title, usually the article title | Same as related `osint.title` |
| `lat` | `number` | Yes | Marker latitude | AOI review coordinate |
| `lon` | `number` | Yes | Marker longitude | AOI review coordinate |
| `observedAt` | ISO datetime string | Yes | Timestamp inherited from `publishedAt` | `2026-06-27T07:44:55.000Z` |
| `confidence` | `number` | Yes | Inherited confidence score | `0.59` |
| `relatedOsintId` | `string` | Yes | Link back to the source `osint` record | `live-osint-google-seoul-airspace-2` |
| `domain` | `string` | No | Publisher/source label | `The Korea Herald` |
| `url` | `string` | No | Source URL | Google News RSS URL |
| `tags` | `string[]` | Yes | Tags plus map-marker provenance | Includes `aoi-review-point` or `global-review-point`, and `not-geocoded` |

## 6. Important Coordinate Policy

| Policy item | Current rule |
|---|---|
| Article geocoding | Not performed |
| OSINT map marker meaning | A review point inside or near the selected AOI |
| Accuracy claim | The app must not claim that `osintEvents.lat/lon` is the exact article/event location |
| Required marker tag | `not-geocoded` |
| Analyst copy guidance | Use wording like “검토 포인트”, “기사 기반 AOI 신호”, or “OSINT review point” |

## 10. How OSINT Is Used in the App

| Consumer | Input | Usage |
|---|---|---|
| Timeline | `osint`, `osintEvents` | Adds article and map-event entries to the chronological signal history |
| Map / globe | `osintEvents` | Draws clickable OSINT markers |
| Fusion Copilot | `osint`, `sourceStatus`, `sourceReasons` | Adds citations, confidence factors, and data-gap context |
| Anomaly logic | `osint.tags`, `confidence` | Creates `osint_correlation` anomalies when relevant activity/security tags exist |
| Briefing | `osint` | Adds source citations and narrative context |

## 11. Planned StealthMole Extension

| Planned field / layer | Intended source | Display rule | Security rule |
|---|---|---|---|
| `stealthmole-cache` source | StealthMole CL/CDS/RM/GM/LM | Add darkweb/leak/ransomware signal cards and optional map markers | Never display raw passwords, user IDs, emails, tokens, or credential values |
| CDS geo signal | `/cds/search?includeGps=true` when authorized | Only show coarse/available `geo` as anonymized threat point | Redact `user`, `password`, `ip`, `username`, `computername` unless explicitly approved for secure backend-only analysis |
| Ransomware / monitoring signal | `/rm/search`, `/gm/search`, `/lm/search` | Display organization/domain-level risk summaries | Treat proof URLs and actor names as untrusted external data |
| Quota/status | `/user/quotas` | Show configured/available modules | Quota calls should not expose credentials or raw JWTs in logs |

## 13. Maintenance Checklist

- [ ] Keep `not-geocoded` on derived OSINT map markers unless real geocoding is implemented and validated.
- [ ] Do not expose API keys or raw JWTs in logs, UI, cache files, or commits.
- [ ] If StealthMole is enabled, store only redacted/aggregated cyber threat intelligence in browser-readable data.
- [ ] Re-run `npm run fetch:live` or the Worker refresh before demos when fresh public OSINT is needed.
- [ ] Verify `sourceStatus` and `sourceReasons` before claiming a source is live.
